# 产品功能设计方案（个人成长陪伴系统）

## 1. 项目概览  

**项目名称**：Personal Growth Companion（PGC）  
**目标**：为用户提供全栈式、可视化、AI‑驱动的成长轨迹管理与建议系统。  
**交付时间**：4 周 MVP 版本，后续迭代持续改进。  

---

## 2. 产品愿景与价值定位  

- **愿景**：让每个人都能随时随地可视化自己的成长路径，获得个性化建议与行为激励。  
- **价值**：  
  1. **全景视角**：知识体系、技能树、家庭关系、财务管理、兴趣爱好多维度同步展示。  
  2. **智能洞察**：大模型多模态分析，提供情绪、学习效率、财务健康等报告。  
  3. **沉浸体验**：3D 渲染让成长轨迹直观可感。  
  4. **团队协作**：家庭成员、团队成员可共享成长数据，互相监督。  

---

## 3. 关键用户角色  

| 角色 | 需求 | 关注点 | 典型行为 |
|------|------|--------|----------|
| **个人成长者** | 跟踪自我成长、获取建议 | 数据可视化、隐私 | 日常记录、查看报告、接受提醒 |
| **家庭成员** | 关注家庭成长、协同规划 | 共享权限 | 查看亲属成长图谱、共同制定目标 |
| **教练/导师** | 监控学员成长、提供辅导 | 监控面板、建议生成 | 评估报告、给出行动建议 |
| **系统管理员** | 保障系统安全与可用 | 访问控制、运维 | 监控日志、配置管理 |

---

## 4. 业务流程图（文字版）  

1. **用户注册/登录** → 账号创建 → 获取 JWT  
2. **数据录入**（知识、技能、家庭、财务、兴趣）  
   - 通过表单或导入（CSV/Excel）完成  
3. **后台处理**  
   - 业务校验 → 存储至对应数据库  
   - 触发消息队列：计划任务、通知、模型推理  
4. **前端展示**  
   - 3D 场景：成长柱状图 + 时间轴  
   - 数据报表：柱形、饼图、折线图  
5. **AI 辅助**  
   - 用户提问 → 调用 gRPC 大模型 → 返回文本/图像建议  
6. **反馈循环**  
   - 用户接受建议 → 记录行动 → 重新评估

## 6. 数据模型概览  

1. **用户表（users）**  
   ```sql
   CREATE TABLE users (
     id uuid PRIMARY KEY,
     username varchar(32) UNIQUE NOT NULL,
     email varchar(128) UNIQUE NOT NULL,
     password_hash varchar(256) NOT NULL,
     role varchar(16) DEFAULT 'user',
     created_at timestamp DEFAULT now()
   );
   ```
2. **知识节点（knowledge）**  
   ```sql
   CREATE TABLE knowledge (
     id uuid PRIMARY KEY,
     user_id uuid REFERENCES users(id),
     title text NOT NULL,
     content text,
     tags jsonb,
     created_at timestamp,
     updated_at timestamp
   );
   ```
3. **技能进度（skills）**（TimescaleDB hypertable）  
   ```sql
   CREATE TABLE skills (
     id uuid PRIMARY KEY,
     user_id uuid REFERENCES users(id),
     skill_name text NOT NULL,
     level int DEFAULT 1,
     updated_at timestamp
   );
   SELECT create_hypertable('skills', 'updated_at');
   ```
4. **家庭关系（family）**（MongoDB 文档）  
   ```json
   {
     "_id": ObjectId,
     "user_id": ObjectId,
     "name": "张三",
     "relation": "父亲",
     "children": [ObjectId, ObjectId],
     "spouse": ObjectId
   }
   ```
5. **财务记录（finance）**  
   ```sql
   CREATE TABLE finance (
     id uuid PRIMARY KEY,
     user_id uuid REFERENCES users(id),
     type varchar(16) CHECK (type IN ('income','expense')),
     amount numeric(12,2),
     category varchar(64),
     created_at timestamp
   );
   ```
6. **文档历史（docs）**  
   ```json
   {
     "_id": ObjectId,
     "user_id": ObjectId,
     "title": "个人成长计划",
     "content_md": "...",
     "version": int,
     "created_at": ISODate
   }
   ```

## 11. 风险评估与缓解  

1. **大模型调用成本**  
   - **缓解**：使用本地推理 + 结果缓存，按需调用。  
2. **多数据库一致性**  
   - **缓解**：采用事件驱动同步，使用 Kafka 作为补偿机制。  
3. **3D 渲染性能**  
   - **缓解**：资源懒加载、LOD（Level of Detail）技术，低端设备降级为 2D 折线图。  
4. **权限泄露**  
   - **缓解**：基于 RBAC 的细粒度授权，JWT 加密存储。

### 5. 安全与合规

| 领域 | 方案 |
|------|------|
| **鉴权** | OAuth2 Authorization Code Grant + JWT，Refresh Token 存于 HttpOnly Cookie。 |
| **权限** | 基于角色的访问控制（RBAC），每个接口在 API 网关层校验。 |
| **加密** | HTTPS/TLS 1.3；数据库字段使用 AWS KMS 加密（如用户邮箱、密码）。 |
| **审计** | 所有写操作均写入审计表（audit_logs），包含 user_id、操作时间、IP。 |

```go
package repository

import (
	"context"
	"database/sql"

	"github.com/jmoiron/sqlx"
	"pgc/auth/internal/model"
)

type UserRepo interface {
	Create(ctx context.Context, u *model.User) error
	FindByUsername(ctx context.Context, username string) (*model.User, error)
}

type userRepo struct {
	db *sqlx.DB
}

func NewUserRepo(db *sqlx.DB) UserRepo {
	return &userRepo{db: db}
}

func (ur *userRepo) Create(ctx context.Context, u *model.User) error {
	query := `INSERT INTO users (username, email, password_hash, created_at)
	          VALUES ($1, $2, $3, $4) RETURNING id`
	return ur.db.QueryRowContext(ctx, query, u.Username, u.Email, u.PasswordHash, u.CreatedAt).Scan(&u.ID)
}

func (ur *userRepo) FindByUsername(ctx context.Context, username string) (*model.User, error) {
	var u model.User
	err := ur.db.GetContext(ctx, &u, `SELECT id, username, email, password_hash, created_at
	                                      FROM users WHERE username=$1`, username)
	if err == sql.ErrNoRows {
		return nil, nil
	}
	return &u, err
}
```

```go
package middleware

import (
	"net/http"
	"strings"

	"github.com/dgrijalva/jwt-go"
	"github.com/gin-gonic/gin"
	"pgc/auth/internal/config"
)

func JWTAuth(secret string) gin.HandlerFunc {
	return func(c *gin.Context) {
		authHeader := c.GetHeader("Authorization")
		if authHeader == "" {
			c.AbortWithStatusJSON(http.StatusUnauthorized, gin.H{"error": "缺少 Authorization Header"})
			return
		}
		parts := strings.Split(authHeader, " ")
		if len(parts) != 2 || parts[0] != "Bearer" {
			c.AbortWithStatusJSON(http.StatusUnauthorized, gin.H{"error": "无效的 Authorization Header"})
			return
		}
		token, err := jwt.Parse(parts[1], func(token *jwt.Token) (interface{}, error) {
			return []byte(secret), nil
		})
		if err != nil || !token.Valid {
			c.AbortWithStatusJSON(http.StatusUnauthorized, gin.H{"error": "Token 验证失败"})
			return
		}
		claims, _ := token.Claims.(jwt.MapClaims)
		c.Set("userID", int64(claims["sub"].(float64)))
		c.Next()
	}
}
```

### 1.5 internal/handler/auth_handler.go  
```go
package handler

import (
	"net/http"
	"time"

	"github.com/dgrijalva/jwt-go"
	"github.com/gin-gonic/gin"
	"golang.org/x/crypto/bcrypt"
	"pgc/auth/internal/model"
	"pgc/auth/internal/repository"
)

type AuthHandler struct {
	UserRepo repository.UserRepo
	JWTSecret string
}

func (ah *AuthHandler) Register(c *gin.Context) {
	var payload struct {
		Username string `json:"username" binding:"required,min=3"`
		Email    string `json:"email" binding:"required,email"`
		Password string `json:"password" binding:"required,min=6"`
	}
	if err := c.ShouldBindJSON(&payload); err != nil {
		c.JSON(http.StatusBadRequest, gin.H{"error": err.Error()})
		return
	}
	hashed, _ := bcrypt.GenerateFromPassword([]byte(payload.Password), bcrypt.DefaultCost)
	user := &model.User{
		Username:     payload.Username,
		Email:        payload.Email,
		PasswordHash: string(hashed),
		CreatedAt:    time.Now(),
	}
	if err := ah.UserRepo.Create(c.Request.Context(), user); err != nil {
		c.JSON(http.StatusInternalServerError, gin.H{"error": "注册失败"})
		return
	}
	c.JSON(http.StatusCreated, gin.H{"message": "注册成功"})
}

func (ah *AuthHandler) Login(c *gin.Context) {
	var payload struct {
		Username string `json:"username" binding:"required"`
		Password string `json:"password" binding:"required"`
	}
	if err := c.ShouldBindJSON(&payload); err != nil {
		c.JSON(http.StatusBadRequest, gin.H{"error": err.Error()})
		return
	}
	user, err := ah.UserRepo.FindByUsername(c.Request.Context(), payload.Username)
	if err != nil || user == nil {
		c.JSON(http.StatusUnauthorized, gin.H{"error": "用户名或密码错误"})
		return
	}
	if err := bcrypt.CompareHashAndPassword([]byte(user.PasswordHash), []byte(payload.Password)); err != nil {
		c.JSON(http.StatusUnauthorized, gin.H{"error": "用户名或密码错误"})
		return
	}
	token := jwt.NewWithClaims(jwt.SigningMethodHS256, jwt.MapClaims{
		"sub": user.ID,
		"exp": time.Now().Add(24 * time.Hour).Unix(),
	})
	signed, _ := token.SignedString([]byte(ah.JWTSecret))
	c.JSON(http.StatusOK, gin.H{"token": signed, "user": gin.H{"id": user.ID, "username": user.Username}})
}

func (ah *AuthHandler) Profile(c *gin.Context) {
	userID := c.GetInt64("userID")
	var user model.User
	if err := ah.UserRepo.FindByID(c.Request.Context(), userID, &user); err != nil {
		c.JSON(http.StatusInternalServerError, gin.H{"error": "查询失败"})
		return
	}
	c.JSON(http.StatusOK, gin.H{"id": user.ID, "username": user.Username, "email": user.Email})
}
```

> **注**：此处 `FindByID` 方法略去实现，可在 `user_repo.go` 添加。  
> **安全**：所有 API 通过 `JWTAuth` 中间件保护，且密码采用 bcrypt 加盐哈希。

```java
    @Autowired
    private KnowledgeService service;

    @PostMapping
    public ResponseEntity<Knowledge> create(@RequestBody @Validated Knowledge kb,
                                           @RequestHeader("X-User-Id") Long userId) {
        return ResponseEntity.ok(service.create(userId, kb));
    }

    @GetMapping
    public ResponseEntity<List<Knowledge>> list(@RequestHeader("X-User-Id") Long userId) {
        return ResponseEntity.ok(service.list(userId));
    }

    @GetMapping("/{id}")
    public ResponseEntity<Knowledge> get(@PathVariable Long id) {
        Knowledge kb = service.get(id);
        if (kb == null) return ResponseEntity.notFound().build();
        return ResponseEntity.ok(kb);
    }

    @PutMapping("/{id}")
    public ResponseEntity<Knowledge> update(@PathVariable Long id,
                                           @RequestBody @Validated Knowledge kb) {
        return ResponseEntity.ok(service.update(id, kb));
    }

    @DeleteMapping("/{id}")
    public ResponseEntity<Void> delete(@PathVariable Long id) {
        service.delete(id);
        return ResponseEntity.noContent().build();
    }
}
```

> **安全**：此 API 通过 `X-User-Id` 头传递用户 ID，实际部署时需配合网关进行 JWT 解析与权限校验。

## 8. 关键业务流程示例（完整）  

1. **用户注册**  
   - 前端 `Login.vue` 发送 POST `/api/auth/register`。  
   - Auth 服务哈希密码后写入 PostgreSQL，返回 201。  

2. **用户登录**  
   - 前端 POST `/api/auth/login`。  
   - Auth 服务校验后返回 JWT，前端存储到 `localStorage`。  

3. **获取知识列表**  
   - 前端 GET `/api/knowledge`，带 `X-User-Id` 头。  
   - Knowledge 服务查询 PostgreSQL 返回 JSON。  

4. **AI 辅助**  
   - 前端调用 gRPC 代理或 REST Gateway（可使用 grpc-gateway）。  
   - AI 服务返回 ChatResponse，前端展示对话。  

5. **前端 3D 渲染**  
   - Vue 组件使用 Three.js 根据知识树数据构建场景。

## 9. 安全与合规细节  

- **JWT**：签名使用 HMAC‑SHA256，私钥存放在 Docker Secrets 或 Vault。  
- **密码**：bcrypt cost 12，定期评估加盐强度。  
- **CORS**：前端仅允许本域或 `*.pgc.com`。  
- **输入校验**：后端使用 Bean Validation / Gin 绑定校验，防止 SQL 注入。  
- **日志**：不记录密码或敏感字段，采用结构化 JSON。  
- **GDPR/CCPA**：提供数据导出（CSV）和删除（Purge）接口，前端可调用。
